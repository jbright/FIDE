REGISTERS

Description

The registers that are available to the programmer.

Register

Logical Name

Description

REGA

BITMASK

Bit mask. The last bit mask specified by the programmer.

REGB

ROMSIG

ROM signature. The last ROM signature specified by the program.

REGC

STSCTL

STS/CTL information. The last status/control information specified by the program.

REGD

BITNUM

Bit Number. The last bit number specified by the programmer in an toggle address, toggle data, or toggle data control function.

REGE

DAT

Data. The last data specified or generated.

REGF

ADR

Address. The last address specified or generated.

REG0

PBDAT

Read probe data. Data accumulated by the Read Probe operation.

REG1-REG7

 

“Local” registers.

REG8-REG9

 

“Global” registers.

  • Registers REGA-REF0 are considered “dedicated” registers and may be updated by specific program statements. However, these registers can be manipulated by the programmer. They are also global.

  • Registers REG1-REG7 are considered “non-dedicated” registers and the programmers can use them at will. They are local the program being executed, and will be erased when an “executed” command is used.

  • Registers REG8-REG9 are also considered “non-dedicated” however, they can be used to pass parameters from one program to another. For example, you can write a generic “delay” routine and use REG8 to pass in the number of “ticks” to delay.
